# Decision Log — PRD PACAL V2

## Session 2026-06-28

### DL-001 — Stockage du code-barres
**Décision :** le code EAN est persisté en base de données sur chaque entrée.
**Raison :** permet l'export CSV, l'affichage dans le détail/PDF, et des enrichissements ultérieurs (CIQUAL, etc.).
**Source :** réponse PAC, Q1.

### DL-002 — Affichage des scores
**Décision :** nutriscore/nova/greenscore n'apparaissent PAS dans la liste historique (vue synthétique). Ils apparaissent dans la vue détail/édition et dans le rapport PDF.
**Raison :** la liste est synthétique, les scores alourdiraient chaque ligne.
**Source :** réponse PAC, Q2.

### DL-003 — Recalcul automatique des kcal
**Décision :** les kcal se recalculent automatiquement quand la quantité change (si des données OpenFoodFacts sont présentes). Si l'utilisateur modifie manuellement les kcal après un calcul auto, la valeur manuelle est conservée et n'est plus recalculée.
**Source :** réponse PAC, Q3.

### DL-004 — Comportement offline / OpenFoodFacts indisponible
**Décision :** le code EAN est conservé dans le formulaire. Un triangle orange (⚠) est affiché sur l'entrée — visible dans l'historique — pour signaler que la fiche est incomplète et invite à la corriger ultérieurement.
**Source :** réponse PAC, Q4.

### DL-005 — Périmètre V2
**Décision :** V2 = scan code-barres + enrichissement OpenFoodFacts uniquement. Stories 2-4 (cible calorique) et 2-5 (alerte déséquilibre) reportées en V2.5. Yuka (2-3) abandonné définitivement.
**Source :** réponse PAC, Q5.

### DL-006 — Yuka abandonné
**Décision :** la recherche de note Yuka est abandonnée. Aucune alternative envisagée.
**Raison :** API non disponible publiquement.
**Source :** réponse PAC dans la spec V2 initiale.

### DL-007 — CIQUAL reporté
**Décision :** l'exploitation des tables CIQUAL est reportée à une version ultérieure (V3 ou Epic 3).
**Source :** réponse PAC dans la spec V2 initiale.

### DL-008 — Mode Fast Path choisi
**Décision :** PRD rédigé en Fast Path (périmètre bien défini, app solo interne, contexte projet connu).

### DL-009 — PRD finalisé
**Date :** 2026-06-28
**Décision :** PRD V2 validé par PAC sans modification. Statut passé à `final`.
